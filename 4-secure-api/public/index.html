Week 4 Security Workflow

Step 1: User Authentication

User logs in and receives a signed JWT token.

Step 2: API Key Validation

Backend checks the x-api-key header before allowing access.

Step 3: Rate Limiting

Repeated failed login attempts trigger brute-force protection.

Step 4: Security Headers

CSP, HSTS, X-Frame-Options, and X-Content-Type-Options are enforced.

Step 5: Monitoring

Winston logs security events and Fail2Ban can monitor failed attempts.
